# HertzQuest LINE 學員系統 — 部署說明

## 系統架構

```
LINE App
  └─ LIFF (register.html)   ← 學員身分登記
  └─ LIFF (event.html)      ← 活動報名（白名單保護）
  └─ 教練後台 (admin.html)  ← 審核 + 活動管理

Google Apps Script (Code.gs) ← 後端邏輯 + LINE 通知
Google Sheets                ← 資料庫（Members / Events / Enrollments）
```

---

## STEP 1 — 建立 Google Sheet

1. 開啟 [sheets.google.com](https://sheets.google.com) → 新建空白試算表
2. 記下網址中的 Sheet ID（`/d/` 和 `/edit` 之間那串）
3. 複製到 `Code.gs` 的 `CONFIG.SHEET_ID`

---

## STEP 2 — 建立 Google Apps Script 專案

1. 開啟 [script.google.com](https://script.google.com) → 新建專案
2. 將 `Code.gs` 的內容貼入預設的 `程式碼.gs`
3. 新增 HTML 檔案（+ 按鈕 → HTML）：
   - 建立 `register`，貼入 `register.html` 內容
   - 建立 `admin`，貼入 `admin.html` 內容
   - 建立 `event`，貼入 `event.html` 內容
4. 填入 `Code.gs` 頂部的 CONFIG 設定值（見下）

---

## STEP 3 — 設定 LINE Developers

### 3-1. 建立 Messaging API Channel
1. 進入 [LINE Developers](https://developers.line.biz/)
2. 建立 Provider → 建立新 Channel → Messaging API
3. 複製 **Channel access token（長期）** → 填入 `CONFIG.LINE_ACCESS_TOKEN`

### 3-2. 建立 LIFF App（共 2 個）
在同一個 Channel 的 LIFF 分頁：

| 用途 | Endpoint URL | 大小 |
|------|-------------|------|
| 學員登記 | `https://script.google.com/.../exec?page=register` | Full |
| 活動報名 | `https://script.google.com/.../exec?page=event` | Full |

複製各自的 **LIFF ID** 填入對應 HTML 檔案：
- `register.html` → `const LIFF_ID = 'YOUR_LIFF_ID_REGISTER'`
- `event.html`    → `const LIFF_ID = 'YOUR_LIFF_ID_EVENT'`

---

## STEP 4 — 部署 GAS 為 Web App

1. GAS 編輯器 → 部署 → 新增部署
2. 類型：**網頁應用程式**
3. 設定：
   - 以誰的身分執行：**我（karta5216295@gmail.com）**
   - 誰可以存取：**所有人**
4. 複製部署後的 URL
5. 填入三個 HTML 檔案中的 `GAS_URL`

---

## STEP 5 — 初始化試算表

在 GAS 編輯器執行 `setupSheets()` 函式一次：
- 自動建立 Members、Events、Enrollments 三個分頁
- 設定表頭和格式

---

## STEP 6 — 設定教練後台 Token

`Code.gs` 的 `CONFIG.ADMIN_TOKEN` 預設為 `hertzquest_admin_2024`，
**請務必改成自己的密碼。**

教練後台網址：`你的GAS網址?page=admin`

## 資料庫結構

### Members（學員表）
| 欄位 | 說明 |
|------|------|
| ID | 系統產生（M + 時間戳記） |
| LINE_UID | LINE 用戶唯一識別碼（白名單核心） |
| 真實姓名 | 學員填寫 |
| 電話 | 學員填寫 |
| 報名課程 | 下拉選單 |
| 指定教練 | 學員填寫 |
| 狀態 | pending / approved / rejected |
| 登記時間 | 自動記錄 |
| 核准時間 | 教練核准時記錄 |
| 備註 | 教練填寫 |

### Events（活動表）
| 欄位 | 說明 |
|------|------|
| ID | 系統產生（E + 時間戳記） |
| 標題 | 活動名稱 |
| 日期 | YYYY-MM-DD |
| 地點 | 活動地點 |
| 說明 | 詳細說明 |
| 名額上限 | 最多可報名人數 |
| 剩餘名額 | 即時更新 |
| 費用 | NT$ |
| 開放報名 | true/false |
| 建立時間 | 自動記錄 |

### Enrollments（報名紀錄表）
| 欄位 | 說明 |
|------|------|
| ID | 系統產生（EN + 時間戳記） |
| 活動ID | 對應 Events.ID |
| 學員ID | 對應 Members.ID |
| LINE_UID | 冗餘欄位，方便查詢 |
| 姓名 | 快照 |
| 電話 | 快照 |
| 狀態 | confirmed |
| 報名時間 | 自動記錄 |

## 發送活動連結

每場活動的報名連結格式：
```
https://liff.line.me/YOUR_LIFF_ID_EVENT?eventId=E1234567890
```

未通過驗證的學員點開後會看到「需要身分驗證」畫面，
通過驗證的學員才會看到「立即報名」按鈕。

## 常見問題

**Q：學員重複提交怎麼辦？**
A：系統會偵測相同 LINE UID，拒絕第二次提交並提示狀態。

**Q：LINE 訊息發不出去？**
A：確認 Channel access token 正確，且 Bot 有加入用戶的好友名單。

**Q：教練可以在 LINE 群組收通知嗎？**
A：設定 `CONFIG.COACH_LINE_GROUP_ID` 為群組 ID（需先將 Bot 加入群組）。
